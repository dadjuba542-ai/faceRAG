# FaceSearch — 单机版会议照片以脸搜图工具

本机运行的会议照片人脸检索工具。批量扫描本地照片、检测人脸、建立特征索引，上传人脸照片即可在历史照片库中找到相似人脸。

**核心原则**：所有数据仅保存在本机，不上传云端，不调用外部 API。

## 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 启动程序

```bash
python run.py
```

或双击 `run.sh` (macOS/Linux) / `run.bat` (Windows)。

程序会自动打开浏览器访问 `http://127.0.0.1:7860`。

### 3. 选择照片目录

在首页输入框填入你的会议照片目录路径（例如 `/Users/xxx/公司会议照片` 或 `D:/公司会议照片`），点击"更换目录"。

**支持格式**：`.jpg` `.jpeg` `.png` `.bmp` `.webp`

### 4. 开始建库

点击"开始建库"，程序会：
1. 递归扫描目录下所有图片
2. 检测每张图片的人脸
3. 过滤过小（< 60px）或低置信度（< 0.5）的人脸
4. 提取人脸特征向量（512维，L2归一化）
5. 保存到 FAISS 索引和 SQLite 数据库

**建库过程不会复制你的原始照片。**

### 5. 搜索人脸

切换到"搜索"标签页：
1. 上传一张包含人脸的查询照片
2. 如果检测到多张人脸，点击选择其中一张
3. 调整 Top K 和相似度阈值（默认 Top 50，阈值 0.35）
4. 点击"开始搜索"
5. 结果按相似度排序，支持打开原图、打开所在文件夹、查看大图

### 6. 增量更新

照片目录新增图片后，点击"增量更新"：
- 只处理新增或修改过的图片
- 已删除的图片标记为缺失
- 不会重复处理未变化的图片

### 7. 重新建库

点击"重新建库"会清空索引和数据库，重新扫描建库。
**不会删除你的原始照片。**

## 配置文件

`config.yaml` 包含所有可调参数：

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `photo_root` | `""` | 照片目录路径 |
| `index.top_k_default` | `50` | 默认返回结果数 |
| `index.similarity_threshold` | `0.35` | 相似度阈值 |
| `index.embedding_dim` | `512` | 向量维度 |
| `face_detection.min_face_size` | `60` | 最小人脸尺寸（px） |
| `face_detection.detection_threshold` | `0.5` | 检测置信度阈值 |
| `thumbnail.save_crop` | `true` | 是否保存人脸缩略图 |
| `thumbnail.crop_size` | `160` | 缩略图尺寸 |
| `thumbnail.jpeg_quality` | `80` | 缩略图质量 |
| `runtime.host` | `127.0.0.1` | 监听地址（请勿改为 0.0.0.0） |
| `runtime.port` | `7860` | 端口 |
| `runtime.auto_open_browser` | `true` | 启动时自动打开浏览器 |

## 数据说明

| 数据 | 存储位置 | 预估大小 |
|------|---------|---------|
| FAISS 索引 | `data/faiss.index` | 100MB–300MB |
| SQLite 数据库 | `data/faces.db` | 几十 MB |
| 人脸缩略图 | `data/face_crops/` | 1GB–5GB（可配置关闭） |
| 日志 | `data/logs/` | 几 MB |

**程序不会复制你的原始照片**，只保存原图路径。

### 备份数据

备份 `data/` 目录即可：
```bash
cp -r data/ data_backup/
```

### 删除本地索引

删除 `data/` 目录即可清空索引和数据库：
```bash
rm -rf data/
```
**不会删除你的原始照片。**

## 常见问题

### Q: 程序启动后浏览器没有自动打开？

手动访问 `http://127.0.0.1:7860`。

### Q: 建库过程中能不能中断？

可以，直接关闭浏览器或按 `Ctrl+C` 中断。已保存的数据不会丢失。下次点"增量更新"继续。

### Q: 搜索结果显示"原图不存在"？

原始照片可能被移动或删除。搜索结果中会有标记。可以点击"重新扫描目录"更新状态。

### Q: 支持 HEIC / TIFF 格式吗？

第一版不支持。后续版本视需求添加。

### Q: 如何升级？

更新代码后重新安装依赖，已有索引可继续使用。如果需要重新建库，点击"重新建库"。

### Q: 如何打包为独立可执行程序？

```bash
pip install pyinstaller
pyinstaller --onefile --name FaceSearch run.py
```

打包前确保所有依赖已安装，建议在 Windows 上打包以获得 `.exe` 文件。

## 技术栈

| 组件 | 用途 |
|------|------|
| Python 3.10+ | 运行环境 |
| InsightFace (ArcFace) | 人脸检测 + 512维特征向量 |

| SQLite | 元数据存储 |
